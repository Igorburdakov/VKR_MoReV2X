# morev2x-highway

Репозиторий для запуска серии экспериментов NR-V2X Mode 2 в highway сценарии. В качестве основы использовал MoReV2X/ns-3 и мобильность из подготовленного SUMO trace для 80 автомобилей.

Карту сделал как овальный highway сценарий из стандарта 3GPP TR 37.885. Единственное поменял прямые участки, чтобы получить плотность около 10-11 UE на lane-km. Овальную форму использовал для того, чтобы автомобили не исчезали на конце прямой дороги и не приходилось дорабатывать появление и исчезновение автомобилей при выходе из карты.

Лицензию проекта взял - GPL-2.0, как и у исходного MoReV2X/ns-3.

## Что внутри

```
morev2x/                              # исходный код MoReV2X/ns-3
assets/highway_oval_80ue_dense/       # SUMO-карта и готовый ns-2 trace на 100 секунд
experiments/run_experiments_final.sh  # итоговый запуск G1, G2, G3
EXPERIMENTS.md                        # описание набора экспериментов
```

## Сборка

Из корня репозитория:

```bash
cd morev2x
./waf configure -d release
./waf build
```

Профиль `release` (`-O3`, без отладочных символов) ускоряет сборку и симуляцию.
Для отладки можно использовать `./waf configure -d debug`.

Если сборка падает из-за зависимостей, обычно не хватает компилятора, Python headers или Boost:

```bash
sudo apt install build-essential python3 python3-dev libboost-all-dev
```

Проверка, что сценарий доступен:

```bash
./waf --run "HIGHWAY_fcd --help"
```

## Запуск всех экспериментов

Из корня репозитория:

```bash
bash experiments/run_experiments_final.sh
```

Скрипт запускает 162 симуляции последовательно: G1 (нумерология), G2 (мощность сигнала) и G3 (RRI отдельно). Результаты сохраняются в:

```
morev2x/results/final_<дата_время>/<метка>/run<N>/
```

Можно запускать отдельные группы:

```bash
bash experiments/run_experiments_final.sh G1
bash experiments/run_experiments_final.sh G2 G3
```

## Как добавить новый эксперимент

Нужно открыть `experiments/run_experiments_final.sh` и добавить строку вида:

```
run <label> <mu> <subch> <rri> <cam> <harq> <tx> <pkeep> <seed>
```

Пример для `mu=1`, `RRI=50`, `CAM=50`, `HARQ=1`, `txPower=20`, `pKeep=0.4`, `seed=1`:

```
run my_experiment  1 50 50 50 1 20 0.4 1
```

Порядок аргументов:

1. `label` - имя папки с результатами.
2. `mu` - нумерология (0, 1 или 2).
3. `subch` - размер subchannel в RB. Для полосы пропускания 20 МГц использовал значения 0 100 RB, 1 50 RB, 2 20 RB. При таких настройках получается один subchannel на slot.
4. `rri` - RRI в миллисекундах.
5. `cam` - период генерации CAM в миллисекундах (использовал для корректировки объема сетевого трафика).
6. `harq` - число передач на один диапазон RRI: 1 или 2 (использовал для корректировки объема сетевого трафика).
7. `tx` - мощность передачи в dBm.
8. `pkeep` - вероятность сохранить ресурс: 0.0, 0.4 или 0.8.
9. `seed` - номер запуска `runNo`, обычно 1, 2 или 3 (необходим чтобы отличать разные симуляции с одной конфигурацией).

Чтобы добавить отдельную группу, нужно добавить новый блок в case:

```
my_group)
echo "=== group ==="
run my_experiment_seed1  1 50 50 50 1 20 0.4 1
run my_experiment_seed2  1 50 50 50 1 20 0.4 2
run my_experiment_seed3  1 50 50 50 1 20 0.4 3
;;
```

Запуск новой группы:

```bash
bash experiments/run_experiments_final.sh group
```
